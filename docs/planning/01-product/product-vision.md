# 제품 비전

## 한 줄 정의

Context-LSP는 AI 코딩 에이전트가 프로젝트의 의도를 배신하지 않도록, 프로젝트 지식 볼트와 Codex 실행 단계를 연결하는 컨텍스트 레이어다.

## 해결할 문제

- 기획, 설계, 구현 현황이 분리되어 프로젝트가 커질수록 의도가 유실된다.
- AI 에이전트가 현재 파일 중심으로 판단해 기존 구조를 무시하거나 없는 제약을 만들어낸다.
- 설계 문서는 작성 후 방치되고 코드와 문서 사이의 drift가 늦게 발견된다.
- 새 프로젝트와 기존 프로젝트는 필요한 컨텍스트 구축 방식이 다르지만 많은 도구가 이를 구분하지 않는다.

## 타깃 사용자

- 개인 개발자와 소규모 팀: Codex/Cursor류 도구로 빠르게 개발하지만 설계 일관성을 잃고 싶지 않은 사용자.
- AI-assisted 오픈소스 프로젝트 관리자: 신규 기여자가 기존 의도와 다르게 코드를 붙이는 일을 줄이고 싶은 관리자.
- 아키텍처 담당자: ADR과 코드 변경의 추적성을 유지하고 싶은 개발 리더.

## 핵심 가치

- 계획 전에 관련 문서와 ADR을 자동으로 모은다.
- 코드 출력 전에 지켜야 할 제약을 명시한다.
- 변경 후 설계 위반과 문서 drift를 finding으로 남긴다.
- 프로젝트의 지식이 사람과 AI 모두에게 재사용 가능한 형태로 축적된다.

## 성공 지표

- Codex가 구현 계획을 작성할 때 관련 ADR과 모듈 문서를 인용한다.
- 새 기능 작업 후 영향을 받은 문서와 코드 경로가 `TraceLink`로 연결된다.
- Brownfield 분석 후 주요 진입점, 테스트 구조, 의존성, 레이어 관례가 문서화된다.
- warning-first 검증으로 사용자가 설계 이탈 근거를 읽고 결정을 내릴 수 있다.

## 비범위

- 모든 언어를 깊이 이해하는 정적 분석기를 v1에 만들지 않는다.
- 모든 설계 위반을 자동 수정하지 않는다.
- 문서가 항상 코드보다 진실이라고 가정하지 않는다.
